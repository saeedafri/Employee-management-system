### CRITICAL ANSWER: Dashboard Identity Problem

The user's core question: "How does the system know which dashboard to show and for which employee?"

**ANSWER (already works, just needs documenting):**

Login response returns:
```json
{
  "user": {
    "id": "user_id",
    "email": "hr@acme.test",
    "memberType": "HR_ADMIN",       ← FRONTEND USES THIS TO ROUTE
    "employeeId": "emp_xyz",        ← USED FOR PERSONAL DATA
    "employee": { ... }
  },
  "accessToken": "eyJ...",          ← STORE IN MEMORY
  "refreshToken": "sessionId.raw",  ← STORE IN httpOnly COOKIE
  "sessionId": "..."
}
```

Frontend routing logic:
```
if memberType == "SUPER_ADMIN" → show Analytics Dashboard (same as HR_ADMIN)
if memberType == "HR_ADMIN"    → show Analytics Dashboard + can see personal data via /employee/dashboard
if memberType == "MANAGER"     → show Manager Dashboard + can see personal data via /employee/dashboard
if memberType == "EMPLOYEE"    → show Employee Dashboard only
if memberType == "AUDITOR"     → show Employee Dashboard (limited)
```

HR_ADMIN personal data: call `GET /employee/dashboard` — uses JWT's `employeeId` automatically.
Manager personal data: call `GET /employee/dashboard` — same.
Both HR_ADMIN and MANAGER also have employee profiles linked in DB.

JWT carries `tenantId` so after login NO X-Tenant-Key header is needed.
