## What was tested

Each country uses **purely data-driven config** — no hardcoded country branches:

1. **Statutory pack** (`StatutoryPack` JSON: tax slabs, contribution schemes, rounding)
2. **Legal entity** (country + currency + work week)
3. **Pay group** (`LITMUS_{code}`) with 100% BASIC component tagged to country wage base
4. **Employee salary** linked to legal entity
5. **OFF_CYCLE payroll run** → `POST /payroll/runs/{id}/calculate` → payslip detail verification

Statutory math validated against the same engine helpers used in production (`computeStatutoryContributions`, `computeIncomeTaxFromRegime`).

## Advisory notes (not failures — UI/backend awareness)

| Item | Detail |
|------|--------|
| **Calculate API body** | `POST /payroll/runs/:id/calculate` requires `{}` body — empty POST returns 400 `INVALID_REQUEST`. |
| **Payslip list vs detail** | `GET /payroll/runs/:id/payslips` returns summary only (no line items). UI must call `GET .../payslips/:payslipId` for earnings/deductions. |
| **Canada federal tax** | Reference litmus expects CA$554/mo; engine returns `WITHHOLDING_TAX` CA$553 (CA$1 rounding on annualized slabs). Net CA$4,067 vs ref CA$4,066 — within tolerance. |
| **Vietnam SI ceiling** | Pack stores `wageCeiling: 4_680_000_000` (×100) because engine applies `/100` minor-to-major on all currencies. Correct SI base = ₫46,800,000 cap. Document this for pack authors. |
| **UAE** | Zero employee deductions (expat model). Gratuity config present but not deducted from net in monthly run. |
| **Tax line code** | Income tax appears as `WITHHOLDING_TAX` on payslip, not `PIT` / `FED_TAX` component codes from pack. |
